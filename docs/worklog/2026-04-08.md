# 작업일지 - 2026-04-08

## 작업 내역

### 아키텍처 결정: Bot Framework + Outgoing Webhook 2버전 병행

Power Automate Workflow 전환을 검토했으나, 스크래핑+LLM+대화상태 관리가 필요한 본 프로젝트에는 부적합하다고 판단. 대신 기존 Bot Framework를 유지하면서 Outgoing Webhook을 병행하는 2버전 구조로 결정.

| 채널 | 엔드포인트 | 용도 | 관리자 승인 |
|------|-----------|------|-----------|
| Bot Framework | `/api/messages` | 1:1 채팅 + Adaptive Card | 필요 (미승인 상태) |
| Outgoing Webhook | `/api/webhook` | 그룹 채팅 + 텍스트 NLU | 불필요 (팀 소유자가 직접 추가) |

### 그룹 채팅 키워드 감지 구현

그룹 채팅에서 @멘션 없이도 예약 의도를 감지하는 `KeywordDetector` 구현.

**감지 레벨:**

| 확신도 | 예시 | 봇 행동 |
|--------|------|---------|
| high | "내일 미팅 하자", "회의실 잡아줘" | 즉시 반응 — NLU 파싱 후 예약 제안 |
| medium | "회의실 3시에 모여" | 조용히 기억 — 나중에 @멘션 시 컨텍스트로 활용 |
| 감지 안 됨 | "내일 점심 뭐 먹지?" | 무시 |

**medium 컨텍스트 활용 흐름:**
```
팀원A: "회의실 3시에 모여"         ← medium, 봇 침묵 (내부 저장, 5분 TTL)
팀원A: "@호비 잡아줘"              ← 모호한 명령 + 저장된 컨텍스트 합침
호비: "오늘 15:00~15:30, 7층 회의실③ 예약할까요?"
```

false positive 방지: "회의록", "회의 끝나", "회의실 어디야" 등 제외 패턴 포함.

**변경 파일:**
- `src/bot/KeywordDetector.ts` — 신규: 키워드 감지 엔진
- `src/bot/ConversationManager.ts` — 그룹 컨텍스트 저장/조회/소비 (saveGroupContext, getGroupContext, consumeGroupContext)
- `src/bot/BookingBot.ts` — 그룹 채팅 분기 추가, @멘션 시 기억된 컨텍스트 합침

## 이슈

- **Teams 관리자 앱 승인 미완**: 관리자가 "아무것도 안 떴다"고 함. Teams 관리 센터(admin.teams.microsoft.com) → 앱 관리 → 보류 중인 요청 확인 필요. 사용자 지정 앱 업로드 정책 자체가 차단되어 있을 가능성 있음.
- **키워드 감지는 Bot Framework 앱 설치 필요**: RSC 권한으로 채널 메시지 수신해야 하므로, 관리자 승인 전까지는 동작 불가.
