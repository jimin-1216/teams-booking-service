# 작업일지 - 2026-04-09

## 작업 내역

### 1. Outgoing Webhook 연결 시도 → 실패

1. `.env`에 `TEAMS_WEBHOOK_SECRET` 추가 완료
2. 서버 재시작 + Cloudflare Tunnel(`bot.sokchokim.com`) 정상 확인
3. Teams 팀(AI개발팀)에 Outgoing Webhook "호비" 등록 완료
   - 콜백 URL: `https://bot.sokchokim.com/api/webhook`
   - Secret 발급 → `.env` 반영
4. 서버 POST 테스트 정상 (HMAC 없이 → `Unauthorized` 응답 확인)

**결과: `@호비` 자동완성 안 됨**
- 원인: 팀 소유자 권한 미보유 — Outgoing Webhook은 소유자만 정상 등록 가능
- 소유자 협조 불가 → **Outgoing Webhook 방식 폐기**

### 2. Power Automate 워크플로우 전환

Outgoing Webhook 대신 Power Automate로 방향 전환.

**서버 변경:**
- `src/bot/WorkflowHandler.ts` — 신규: Power Automate용 `/api/workflow` 엔드포인트
- `src/index.ts` — 워크플로우 핸들러 등록

**Power Automate 플로우 구성:**
1. 트리거: "When someone reacted to a message in chat" (꿀벌 이모지 🐝)
2. 액션1: Get message details (MessageId → 메시지 본문 조회)
3. 액션2: HTTP POST → `https://bot.sokchokim.com/api/workflow`
   - Body: `{ text: plainTextContent, userId, userName }`
4. 액션3: Post message in a chat or channel (응답을 채널에 회신)

**테스트 결과:**
- 서버 `/api/workflow` 엔드포인트 정상 동작 확인 (curl 테스트 성공)
- NLU 응답 정상: "내일 오후 3시에 7층 회의실 예약해줘" → 회의실 추천 응답 확인
- 플로우 트리거 → HTTP 성공 → **Post message 단계에서 실패**
  - 원인: HTTP 응답이 JSON 객체(`{"reply":"..."}`)인데, Post message는 String/html 타입 필요
  - 해결: Parse JSON 액션 추가하여 `reply` 값만 추출 필요 (진행 중)

## 이슈

### Outgoing Webhook 소유자 권한 문제
- 비소유자가 등록해도 활성화 안 됨 → 폐기

### Power Automate 트리거 제약
- "When someone reacted to a message in **chat**" → 채팅에서만 동작, 채널은 별도 확인 필요
- 채널용 리액션 트리거 존재 여부 미확인

### Post message 타입 에러
- HTTP 응답 JSON을 Parse JSON으로 파싱 후 `reply` 문자열만 전달해야 함

### 3. E2E 테스트 성공

- Parse JSON 액션 추가 → `reply` 값만 Post message에 전달 → 성공
