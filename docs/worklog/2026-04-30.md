**문제**: 창업센터에서 일 3시간 이상 예약 시 퇴거 조치. 사용자가 현황을 모른 채 예약하면 위험.

**해결**: 예약 추천 시 항상 누적/잔여/예약 후 잔여 시간을 표시하도록 변경.

- `src/bot/NLUHandler.ts` — dailyUsage를 항상 조회하여 추천에 포함
- `src/bot/BookingBot.ts` — 카드 기반 예약에도 사전 안내 메시지 추가

### 2. 얍삽이 모드 (완료)

**문제**: 3시간 초과 사용이 필요할 때 다른 사람이 그 시간을 선점하면 곤란.

**해결**: 초과분을 팀원 이름으로 자동 분할 예약.

변경 파일:
- `src/rules/BookingSplitter.ts` — 신규. 분할 로직 + 미리보기 메시지
- `src/config.ts` — `sneakyMode` 설정 블록 추가
- `src/bot/ConversationManager.ts` — `sneakySplit` 상태 필드 추가
- `src/bot/NLUHandler.ts` — 정책 위반 시 얍삽이 모드 분기, 분할 예약 순차 실행

설정:
```env
SNEAKY_MODE=true
SNEAKY_MEMBERS=홍길동,김철수,이영희
```

### 3. Teams SDK v2 마이그레이션 조사 (완료)

**조사 결과**:
- 블로그: https://microsoft.github.io/teams-sdk/blog/bring-your-agent-to-teams/
- 패키지: `@microsoft/teams.apps` v2.0.8 (2026-04-16 릴리즈, MIT)
- 핵심: "Your server is yours" — Express 서버에 어댑터만 끼우는 방식
- 배포: `teams app create` CLI로 AAD 등록/매니페스트 자동 생성

**현재 방식 vs 새 SDK**:
| 항목 | Bot Framework v4 | Teams SDK v2 |
|------|-------------------|--------------|
| AAD 등록 | Azure Portal 수동 | CLI 자동 |
| 매니페스트 | JSON 수작업 | CLI 자동 생성 |
| 보일러플레이트 | ~100줄 (Adapter, Auth, Handler) | ~20줄 (이벤트 리스너) |
| 의존성 | botbuilder, botframework-connector | @microsoft/teams.apps |

**마이그레이션 범위**: `BookingBot.ts` + `index.ts` 교체. 나머지 비즈니스 로직은 그대로.
