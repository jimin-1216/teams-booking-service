### 2. Container Apps 스케일링 이슈 해결

- **증상**: Teams에서 예약 시도 → 공실 조회까지는 됐으나 실제 예약 미완료
- **원인**: KEDA가 트래픽 없을 때 `Replicas: 0`으로 스케일다운 → 콜드스타트(이미지 pull 89초) 중 타임아웃
- **해결**: `az containerapp update --min-replicas 1` 설정
- **리소스 그룹**: `cg-teams` (기존 `rg-cginside`에서 변경됨)

### 3. 예약 실행 버그 수정 (로그 분석 기반)

서버 로그 분석으로 3가지 버그 발견 및 수정:

| 버그 | 원인 | 수정 |
|------|------|------|
| 폼 열기 버튼 오감지 | invisible disabled 버튼(x:0)이 `x < 830` 통과 | `y < 150` + visible 체크로 변경 (`2d2efe6`) |
| 제출 시 엉뚱한 버튼 클릭 | 제출 버튼 disabled → fallback이 폼 열기 버튼 클릭 | `y > 400` 제한 + fallback 제거 + 활성화 대기 5초 (`2d2efe6`) |

## Teams + Azure 실패 분석

### 근본 원인: 로컬-서버 환경 격차

| | 로컬 (성공) | 서버 (실패) |
|--|------------|-----------|

| OS | Windows 11 | Linux (Debian/Jammy) |
| 네트워크 | 한국 직결 | Azure eastasia (홍콩) |
| 리소스 | 충분 | 1CPU / 2GB 제한 |
| 디버깅 | 브라우저로 눈으로 확인 | 로그/스크린샷만 |

### 왜 Teams를 포기하는가

1. **Azure 강제 종속**: Teams 봇은 Azure Bot Service + Azure AD가 필수. 다른 클라우드/셀프호스팅 불가
2. **디버깅 사이클이 치명적**: 코드 수정 → push → CI/CD 빌드(3분) → 배포(2분) → Teams 테스트 → Cloud Shell 로그 확인. 한 번 고치는데 10분+
3. **Azure for Students 제약**: 리전 제한(koreacentral 불가), 리소스 제공자 미등록, 크레딧 소모
4. **인증 복잡도**: AppID + AppPassword + TenantID + SingleTenant 설정. 비밀 만료/불일치 이슈 반복
5. **headless 환경 차이 해소 불가**: 로컬에서 아무리 고쳐도 서버에서 다르게 동작. React SPA 스크래핑의 타이밍/렌더링 차이를 원격으로 디버깅하는 건 비현실적
