## 현황 점검

### 현재 동작 상태
- Power Automate 워크플로우 연결 성공 (2026-04-09)
- 트리거: Teams 채팅에서 메시지에 🐝 리액션
- 흐름: 리액션 → Power Automate → HTTP POST `/api/workflow` → 서버 NLU 응답 → Teams에 포스트
- 1턴 요청-응답은 정상 동작 확인됨

### 핵심 문제: 대화 핑퐁 불가

Power Automate 워크플로우가 단발성(리액션 → HTTP → 포스트) 구조라서 **멀티턴 대화가 불가능**.

서버에는 `ConversationManager`로 대화 상태 관리 코드가 완성되어 있으나, 사용자의 후속 응답("응", "취소")을 서버로 다시 전달할 수단이 없음.

**이것이 문제인 이유:**
- 이 프로젝트의 핵심 목적은 마일 UI의 비효율 해소 (공실 확인 → 비교 → 선택 → 예약)
- 사용자가 추천 결과를 보고 확인/변경/취소를 선택하는 흐름이 필수
- 일 3시간 합산 사용량 확인도 예약 전에 보여줘야 함
- 1턴 완결(바로 예약)로는 이 문제를 해결할 수 없음

### Teams 연결 방식별 현황

| 방식 | 상태 | 차단 사유 |
|------|------|-----------|
| Bot Framework 앱 | 차단 | 관리자 승인 안 뜸 (앱 제출/업로드 문제 추정) |
| Outgoing Webhook | 폐기 | 팀 소유자 권한 미보유 |
| Power Automate (현재) | 제한적 | 단발성 — 멀티턴 대화 불가 |

## 해결 방안: Adaptive Card + Wait

Power Automate에 **"Post adaptive card and wait for a response"** 액션 활용.

일반 메시지 포스트와 달리:
- Adaptive Card(버튼/선택지 포함)를 Teams에 포스트
- **사용자가 버튼을 누를 때까지 플로우 실행이 대기**
- 사용자 클릭 후 플로우가 이어서 실행 (추가 HTTP 호출 가능)

### 예상 흐름

```
사용자: "내일 3시 회의실" + 🐝 리액션
  ↓
Power Automate 트리거
  ↓
HTTP POST → 서버 /api/workflow (NLU 파싱 + 공실 조회)
  ↓
서버 응답: { type, rooms[], date, startTime, endTime, dailyUsage }
  ↓
Power Automate: Adaptive Card 포스트 + WAIT
  ┌─────────────────────────────────┐
  │ 내일 15:00~15:30                │
  │ 추천: 7층 회의실 ①              │
  │ 오늘 사용량: 1.5 / 3시간        │
  │                                 │
  │ [예약] [다른 회의실] [취소]      │
  └─────────────────────────────────┘
  ↓ (사용자 버튼 클릭 대기)
사용자: "예약" 클릭
  ↓
HTTP POST → 서버 /api/workflow/confirm (예약 실행)
  ↓
결과 메시지 포스트
```

### 서버 변경 필요 사항

1. `/api/workflow` 응답을 plain text → **구조화된 JSON**으로 변경
   - `{ type: "recommendation", rooms: [...], date, startTime, endTime, dailyUsage }`
   - Power Automate가 이 데이터로 Adaptive Card 조립

2. `/api/workflow/confirm` 엔드포인트 신규 추가
   - `{ action: "book"|"cancel", roomId, userId, userName }`
   - 예약/취소 실행 후 결과 반환

3. 취소 흐름도 동일 구조로 처리 가능
